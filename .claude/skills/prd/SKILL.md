# PRD — Product Requirements Document

Skill que contiene los requisitos del producto. Consultarlo antes de implementar features nuevas o tomar decisiones de arquitectura.

## Cuándo usar este skill

- Antes de implementar una feature nueva, para verificar que se alinea con los requisitos
- Cuando haya dudas sobre el alcance o prioridad de una funcionalidad
- Para entender el contexto del producto al hacer code reviews o refactors

## Archivos

- `requirements.md` — PRD completo del proyecto

## Instrucciones

1. Lee `requirements.md` para entender el contexto
2. Valida que la tarea actual se alinea con los objetivos y funcionalidades descritas
3. Si la tarea contradice el PRD, notifica al usuario antes de proceder
4. Respeta las prioridades definidas en el PRD al sugerir implementaciones
